# Panaconda: Pangenome GEXF Graph Layout

This repository contains the graph layout engine used to generate visualizations for pangenomes and synteny graphs. It leverages the Gephi Toolkit to apply customized, multi-level force-directed layouts (such as Yifan Hu) to `.gexf` instantiations of genome graphs.

This software is part of the methodology described in the following pan-synteny graph paper:

> **Panaconda: Application of pan-synteny graph models to genome content analysis**  
> Andrew S. Warren, James J. Davis, Alice R. Wattam, Dustin Machi, João C. Setubal, Lenwood S. Heath  
> *bioRxiv* 215988; doi:[https://doi.org/10.1101/215988](https://doi.org/10.1101/215988)

## Overview

Traditional whole-genome alignments often struggle to represent the complex architectural conservation and changes across multiple prokaryotic genomes. Panaconda utilizes pan-synteny graphs to capture these relationships. 

This repository handles the **layout step** of the pipeline. It reads raw, unpositioned synteny graphs generated by Panaconda in GEXF format, applies a highly optimized multi-level coarsening and physical layout algorithm (handling thousands of nodes/edges efficiently), and exports the fully positioned graph for rendering.

## Requirements
* Java 11 or higher
* `gephi-toolkit.jar` (Included in `lib/`)

## Compilation 

To compile the source code natively and create a modern, Java 11-compatible single executable JAR (Fat JAR), run the following commands from the project root:

```bash
# 1. Compile the java files
mkdir -p build_classes
javac -cp "lib/gephi-toolkit.jar" $(find src -name "*.java") -d build_classes

# 2. Extract the Gephi Toolkit into the build folder
cd build_classes
unzip -qn ../lib/gephi-toolkit.jar

# 3. Remove legacy security signatures from the toolkit
rm -rf META-INF/*.SF META-INF/*.DSA META-INF/*.RSA META-INF/ECLIPSEF.RSA

# 4. Package into a single executable JAR
cd ..
echo "Main-Class: layout.ImportExport" > manifest.txt
jar cfm bin/gexf_layout.jar manifest.txt -C build_classes .
